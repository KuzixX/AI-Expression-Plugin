# Motion Groups Transfer — версия 1

Скрипт для **топология-независимого переноса локальных деформаций** (translation + rotation + stretch) между двумя head-мешами через **разбиение на motion-группы**.

Файл: [`multi_anchor_motion_groups.py`](./multi_anchor_motion_groups.py)

---

## Назначение

Дано:
- **Голова 1** (source) — FLAME-меш с известным блендшейпом (экспрессией)
- **Голова 2** (target) — может быть FLAME (другая форма) или **произвольный FBX** с другой топологией

Задача: перенести деформацию (мимику) с головы 1 на голову 2, сохранив локальные паттерны движения, а не только общий "средний сдвиг" зоны.

---

## Идея метода

Классический ws-перенос даёт **одну усреднённую стрелку** на всю нагретую зону. Если внутри зоны есть антагонистические движения (верхняя губа вверх + нижняя вниз), они **сокращаются** в среднем → перенос даёт ноль.

**Motion-groups решение:** разбиваем зону на под-группы где **движение более-менее однородно**. Для каждой под-группы считаем полное **полярное разложение** (translation + rotation + stretch). Переносим эти трансформации индивидуально.

---

## Алгоритм пошагово

### Шаг 0 — Подготовка

```
verts_1, faces_1  = FLAME(shape_1)        — голова 1
verts_2, faces_2  = FBX или FLAME(shape_2) — голова 2
```

Обе нормируются в bbox=1.0 (центр в нуле, диагональ = 1).

### Шаг 1 — Heat diffusion из anchor-точек

Пользователь выбирает по N (до 20) **anchor-точек** на каждой голове через Shift+клик. На голове 2 нужно ровно столько же сколько на 1й — в том же анатомическом порядке (нос→бровь→...).

Для каждой anchor-точки решаем дискретное уравнение теплопроводности:

```
(A - t·Lc) u_new = A · u_old
```
где `A` — mass matrix (Voronoi-площади вершин), `Lc` — cotangent Laplacian, `t` — время диффузии.

**Начальное условие:** `u[src] = 1 / A_src` — это **mass-normalized δ-функция**. Полная масса тепла = 1 на обеих мешах (закон сохранения), что делает диффузии сравнимыми между мешами с разной топологией.

**Результат:** `heat_1[a, v]`, `heat_2[a, v]` — heat-поля от каждой anchor-точки на каждой голове.

### Шаг 2 — Применение блендшейпа к голове 1

```
head1_expr = FLAME(shape_1 + expr_betas)
δ_1 = head1_expr - verts_1
```

`δ_1` — это **истина** мимики на голове 1. Дальше мы её разбиваем и переносим.

### Шаг 3 — Motion-groups на голове 1

Для каждой anchor-зоны независимо:

#### 3.1. Отсев холодных вершин
```
active = { v : heat_1[a, v] > 5% · max(heat_1[a]) }
```
Это **geodesic-aware** фильтр (тепло следует поверхности).

#### 3.2. K-means в 6D feature space

Каждая активная вершина имеет 6-мерный вектор:
```
feature[v] = [
    δ_1[v] / motion_scale,                          # 3D — направление движения
    (verts_1[v] - mean) / pos_scale · position_weight  # 3D — положение
]
```

`position_weight=1.5` балансирует "похожесть движения" vs "пространственная локальность". Это **приближение геодезической константы**: вершины далеко по xyz не попадут в одну группу даже если δ похожи.

K-means с адаптивным `n_clusters` (max 5 по умолчанию).

#### 3.3. Полярная декомпозиция в каждой группе

Для каждой motion-group берём её вершины и считаем **взвешенный деформационный градиент F (3×3)**:

```
F = (Σ w · q ⊗ p) · (Σ w · p ⊗ p)⁻¹
```
где `p = r - c_rest` (centered rest), `q = (r + δ) - c_def` (centered deformed), `w` — heat-веса.

**Полярная декомпозиция через SVD:** `F = U Σ V^T`
- **R = U · V^T** — поворот (ортогональная, det = +1)
- **S = V · Σ · V^T** — растяжение (симметричная PSD)
- `stretches = eig(S)` — коэф. растяжения по 3 главным осям

**Дескриптор кластера** (полностью топология-независимый):
```python
{
  'anchor_idx':    int,          # к какой anchor-зоне относится
  'c_rest':        (3,),         # heat-weighted центроид в норм. пространстве
  'spatial_sigma': float,        # RMS-радиус кластера (для geodesic-лимита)
  'μ':             (3,),         # translation
  'R':             (3, 3),       # rotation matrix
  'S':             (3, 3),       # stretch tensor
}
```

Никаких индексов вершин — описание чисто геометрическое + трансформационное.

### Шаг 4 — Voronoi-разбиение зон головы 2

Для каждой anchor-зоны на голове 2:

#### 4.1. Активные вершины
```
active_2 = { v : heat_2[a, v] > 5% · max(heat_2[a]) }
```
Опять heat-based geodesic фильтр.

#### 4.2. Voronoi + spatial-σ лимит

Для каждой активной вершины:
```
для каждого source-кластера k этого же anchor'а:
    dist_k = ||verts_2[v] - c_rest_source_k||                # Euclidean
    if dist_k > σ_source_k · geodesic_factor:
        dist_k = ∞                                            # вне досягаемости

nearest = argmin(dist_k)                                       # ближайший
if all distances == ∞:
    skip v                                                     # не получит деформацию
```

**`geodesic_factor=3.0`** (default) — даёт 3σ-rule по spatial spread кластера. Можно подкрутить:
- `1.5` — строго (только плотная часть)
- `3.0` — мягко
- `∞` — без лимита

### Шаг 5 — Применение трансформаций

Для каждого target-кластера (с матчем к source):

```
c_target = heat-weighted центроид target вершин
RS_source = R_source · S_source                              # deformation gradient

для каждой target-вершины v в этом кластере:
    r = verts_2[v] - c_target                                # offset от центра
    δ_2[v] += heat_2[v] · ( μ_source + (RS_source - I) · r )
    weight_total[v] += heat_2[v]
```

После всех кластеров: `δ_2[v] /= weight_total[v]` — нормировка для вершин в перекрывающихся зонах anchors.

### Шаг 6 — Laplacian smoothing

Грубое cluster-разбиение даёт **резкие границы** между группами. Сглаживаем:

```
W = neighbor-averaging sparse matrix (по рёбрам меша)
для каждой итерации:
    δ_2 = (1 - α) · δ_2 + α · (W @ δ_2)
```

Default: `--smooth-iters 50 --smooth-alpha 0.5` — мягкое смешивание.

### Шаг 7 — Финальная деформация

```
head2_def = verts_2 + δ_2
```

## Параметры (CLI)

| Флаг | Default | Что делает |
|---|---|---|
| `-n` | 20 | Макс. число anchor-точек на голове 1 |
| `--time` | спрашивает | Время диффузии (0.0005=локально, 0.002=средне, 0.01=широко) |
| `--steps` | спрашивает | Шагов implicit Euler (60 — хорошо) |
| `--n-clusters` | 5 | Макс. число motion-groups в каждой зоне |
| `--heat-threshold` | 0.05 | Доля от heat.max ниже которой вершина игнорится |
| `--position-weight` | 1.5 | Вес позиции vs motion в k-means feature space |
| `--geodesic-factor` | 3.0 | Радиус досягаемости кластера = σ × этот множитель |
| `--smooth-iters` | 50 | Кол-во Laplacian smoothing итераций |
| `--smooth-alpha` | 0.5 | Сила сглаживания на итерацию (0..1) |
| `--fbx` | None | Путь к FBX для головы 2 (вместо FLAME-формы) |
| `--fps` | 24 | FPS для анимации диффузии |

## Что показывает в окнах

1. **Окно диффузии** — анимация распространения тепла со всех anchor-точек одновременно. Q закрывает.

2. **Окно morph** — обе головы плавно деформируются rest → deformed. Цвет = карта magnitude смещения.

3. **Окно motion-groups (последнее)** — обе головы в deformed-состоянии с раскраской по cluster-membership:
   - **Слева:** голова 1 (source) — резкие границы между кластерами
   - **Справа:** голова 2 (target) — границы сглажены теми же smoothing-итерациями что и δ_2

   **Совпадение цветов = соответствующие анатомические зоны.**

## Сильные стороны метода

- **Топология-независимый**: дескриптор кластера = чистая геометрия + трансформация, без индексов вершин
- **Захватывает не только translation**: rotation и stretch учитываются явно через полярную декомпозицию
- **Локальные паттерны сохраняются**: разные кластеры в одной зоне → разные направления движения (антагонисты)
- **Размер дескриптора компактный**: ~25 чисел на кластер, не зависит от N вершин

## Ограничения

- **Euclidean подмена геодезика**: `geodesic_factor · σ` использует xyz-расстояние, не путь по поверхности. На мешах со складками (Metahuman веки, носогубные борозды) могут быть ошибки.
- **Greedy Voronoi**: каждая target-вершина приписывается к одному кластеру (с soft cap'ом через сглаживание). Без формального оптимального matching.
- **Анатомическое выравнивание неявное**: anchor-точки пользователь ставит руками. Если они в разных анатомических местах — матчинг неправильный.
- **K-means недетерминирован**: разные запуски могут дать разные кластеры (фиксированный `random_state=0` минимизирует, но не убирает).
- **Без LBS**: метод выдаёт δ-поле на меше, а не веса для системы костей.
