# Motion Groups — версия **6.0** (HEAD 1 + опц. FBX heat-зоны)

После v5 проведён радикальный strip: **убран матчинг голов и перенос
деформаций**. v6 — чистый зональный анализ. Основа — source-голова FLAME
(HEAD 1, с деформацией от экспрессии). Дополнительно можно подгрузить
**произвольную FBX-голову** и прогнать на ней **тот же heat/зональный
анализ независимо** (без матчинга и переноса) — у FBX нет модели экспрессии,
поэтому для неё считаются только heat-зоны (кластеризация по heat+позиции).

## Что осталось (перенесено из v5)

- Пресеты настроек (save/load в `presets/`)
- Форма головы (shape betas preset)
- Экспрессия (expression preset + custom betas)
- Heat-диффузия (+ авто-стоп при overlap зон)
- Кластеризация (`kmeans` / `agglomerative`)
- Сглаживание (Laplacian δ-smoothing)
- Anchor-точки (Shift+click pick)
- Multi-t enrichment (+ маскировка single-t reach'ем)

   decorr_heat / ring_match / tps_global / direct_copy / flame_fit), матчинг

## FBX-голова (опционально)

Если в GUI указан путь к FBX (`fbx_path`), после анализа HEAD 1 запускается
**отдельный, независимый** прогон на FBX:

1. FBX → OBJ через `assimp export`, загрузка trimesh, `normalize_bbox`.
2. Выбор anchor-точек (Shift+click) на FBX.
3. Диффузия + (опц.) multi-t enrichment + single-t маскировка — как у HEAD 1.
4. **Кластеризация по heat+позиции** (δ=0, т.к. нет модели экспрессии).
5. Окна: только **ОКНО 1 (диффузия)** и **ФИНАЛ (зоны, одиночный меш)**.
   Окна rest|deformed и native|recon|smoothed пропускаются (нет δ).
   μ-стрелки не рисуются (нет деформации) — только шары на anchor'ах.

Дампы FBX → `run_v6_*/fbx/` (без `delta_*.csv`).

## Пайплайн

1. **Выбираем anchor-точки** (heat sources) — Shift+click.
2. **Диффузия** тепла от anchor'ов (с опц. авто-стопом при пересечении зон).
3. **Кластеризация:**
   - `multi_t` ВЫКЛ → кластеризуем **всю голову** per-anchor по single-t
     heat-порогу.
   - `multi_t` ВКЛ  → кластеризуем **только внутри зоны ответственности**
     каждого anchor'а (hard argmax-partition), маскированной single-t reach'ем.
4. **Финальное окно:**
   - `multi_t` ВКЛ → замаскированные hard-partition зоны (цвет = anchor).
   - `multi_t` ВЫКЛ → кластеры, окрашенные смесью single-t (цвет =
     доминирующий по heat кластер).

> Без матчинга голов и без переноса деформаций.

## UV-развёртка masked-зон (только multi-t)

При включённом `multi_t` после финального окна открывается **новое окно
сравнения UV-развёрток** всех голов:

1. Берём masked hard-partition зоны (каждая зона = anchor).
2. Каждую зону разворачиваем **одним островом** (без доп. сегментации):
   - оставляем крупнейшую связную компоненту зоны (мелкие несвязные ошмётки
     argmax-разбиения отбрасываем);
   - **режим разворота** (настройка `uv_flat` в секции *UV settings*):
     - `uv_flat = False` (по умолчанию) — Tutte-инициализация (граница →
       квадрат, гармонический solve) + ARAP-релакс (Liu 2008): натуральная
       форма, минимум искажений;
     - `uv_flat = True` — планарная проекция на плоскость best-fit (PCA),
       как режим **Flat** в Cinema4D (быстро, но возможны перехлёсты на
       загибах);
     - `uv_world_orient = True` (режим **world-flat**) — планарная проекция
       вдоль НОРМАЛИ зоны с фиксированной мировой ориентацией: мировой Y → +V
       (вверх в UV), (n×up) → +U. Базис берётся из мировых направлений, а не из
       формы зоны → ориентация **детерминированная** и одинаковая на всех
       головах (обе в общем `normalize_bbox`-кадре), `_orient_uv_canonical` не
       применяется. Для почти горизонтальных зон (темя) — fallback на мировой Z.
       Имеет приоритет над `uv_flat`.
   - **каноническая ориентация** (для arap/flat): мировое «вверх» (Y) → +V,
     «вправо» (X) → +U (с фиксацией отражения). Обе головы нормализованы в одном
     фрейме → одинаковые зоны развёрнуты единообразно и совпадают при сравнении.
3. Раскладываем зоны по **сетке ячеек** внутри единичного квадрата.
   Зона `a` всегда попадает в одну и ту же ячейку → острова одной зоны у
   разных голов лежат в одном месте → прямое сравнение.
4. Развёртки голов показываются рядом (цвет острова = anchor, серая сетка
   ячеек).
5. **3-я панель — overlay**: соответствующие зоны обеих голов наложены в одни
   и те же ячейки общей сетки. HEAD 1 — заливкой (тёплый цвет), 2-я голова —
   каркасом (холодный цвет) поверх. Позволяет напрямую сравнить форму
   развёрнутых зон. При `uv_overlay_scale_match=True` (по умолчанию) остров
   2-й головы подгоняется по масштабу и центру к острову 1-й головы той же
   зоны (совпадение центроида + **габаритного радиуса**, 97-й перцентиль
   расстояния до центроида) — подгонка симметрична: больший остров сжимается,
   меньший растёт, и 2-я голова не торчит за 1-ю (RMS-радиус зависел бы от
   формы и мог раздуть остров другой формы). При `False` остров кладётся как
   есть (каждый нормирован по своему bbox в ячейку).
6. **4-я панель — перенесённые группы** (только если включён перенос на FBX):
   развёртка FBX, раскрашенная **перенесёнными группами** (global cluster id
   из FLAME, per-vertex) — наглядно видно, какие FLAME-группы легли на какие
   участки FBX. Непокрытые вершины — серые.

**Boundary heat-warp** (`uv_warp_heat`, по умолчанию выкл): нежёсткая подгонка
острова 2-й головы (FBX) к острову FLAME перед UV-NN. Берём граничные вершины
острова FBX, тянем каждую к **ближайшей точке на граничной ЛОМАНОЙ FLAME**
(проекция на отрезки границы, не на вершины — `uv_warp_heat` ⇒ плавное
соответствие). Внутренние точки следуют за границей по **тепловому полю
влияния**: решаем неявное тепло `(MM + t·L)·H = MM·E` (E — one-hot по границе) в
UV-домене острова, нормируем строки в partition-of-unity; граничные строки
пиним в one-hot, чтобы граница села **точно** на цель. Смещение внутренней
вершины = `W @ disp_boundary`. Параметр `uv_warp_heat_t` (default 0.05) — время
диффузии (больше = более гладкое/глобальное влияние). Альтернатива PCA+ICP при
сильно разной форме развёртки; влияет на overlay и точность UV-NN переноса.

Параметр `uv_warp_min_dist` (default 0.0) — **максимальный зазор** «точка → ребро»
после подгонки. При `0` граница садится точно на ребро (residual 0). При `> 0`
точки, оказавшиеся дальше порога, подтягиваются ровно до `min_dist` от ребра, а
уже более близкие — не трогаются (итог: расстояние ≤ `min_dist` везде). Тот же
кэп применяется и к зелёным линиям связи на overlay-панели.

При `uv_warp_line_step > 0` на **3-й (overlay) панели** окна «UV-развёртка
masked-зон» поверх наложенных островов рисуются **зелёные линии связи**:
каждая граничная точка острова FBX → её проекция на ребро границы острова
FLAME. Линии строятся **в системе УЖЕ подогнанных островов** overlay-панели
(после scale-fit / PCA+ICP), поэтому показывают фактическую тягу границы.
`uv_warp_line_step` задаёт шаг показа линий (0 = выкл, 1 = все, 2 = через одну,
6 = каждая 6-я и т.д.) — чтобы не загромождать картинку. Отдельного окна нет.

При включённом `uv_warp_heat` overlay-панель показывает остров FBX **уже
деформированным** (heat-warp применяется прямо в overlay): граница FBX садится
на ребро FLAME, нутро следует за тепловым полём. Поэтому наложенные шеллы
совпадают по краю, а зелёные линии связи схлопываются в ~0 (или ≤ `uv_warp_min_dist`).
Без `uv_warp_heat` остров FBX лишь масштабируется (scale-fit), и линии показывают
оставшийся зазор до ребра.

**PCA+ICP выравнивание** (`uv_align_pca_icp`, по умолчанию выкл): дополнительно
к канонической ориентации + scale-fit подгоняет остров 2-й головы к острову
1-й головы той же зоны similarity-преобразованием — PCA по главным осям
(перебор 4 знаков, лучший по NN-дистанции) + ICP-уточнение (NN-соответствия →
Umeyama-similarity). Работает при разной триангуляции/числе вершин. Влияет и на
overlay-панель (3-я), и на **точность UV-NN переноса** δ/групп на FBX.

Реализация на numpy/scipy (внешние UV-библиотеки не нужны).

## Перенос деформаций FLAME → FBX (UV-NN, только multi-t + FBX)

После UV-окна, если включён `multi_t`, задан FBX и `transfer_to_fbx=True`,
деформации переносятся с FLAME (HEAD 1) на FBX по **UV nearest-neighbor**:

0. **Сопоставление зон по позиции** (не по индексу пика!). Обе головы прошли
   `normalize_bbox` → лежат в одном кадре. Зону FBX паруем с зоной FLAME по
   **ближайшему центроиду** зоны (one-to-one, венгерский алгоритм
   `scipy.optimize.linear_sum_assignment`). Так глаз садится на глаз, нос на
   нос, даже если anchor'ы на двух головах кликнуты в разном порядке (Open3D
   `get_picked_points()` порядок кликов не гарантирует). Это же сопоставление
   используется и в UV-окне: острова одной зоны ложатся в одну ячейку и
   красятся одинаково на всех панелях.
1. Для каждой зоны, общей обеим головам, берём её UV-остров на каждой голове
   (та же параметризация, что и для UV-окна) и нормируем оба острова в общий
   фрейм (центроид + RMS-радиус) — благодаря канонической ориентации зоны
   совпадают.
2. Для каждой вершины FBX в зоне ищем **ближайшую вершину FLAME в UV**
   (`scipy.spatial.cKDTree`) и копируем её `δ` (деформацию), global cluster id
   и номер зоны.
3. Применяем `δ` к rest-вершинам FBX → deformed FBX. Непокрытые вершины
   (вне зон / откинутые мелкие куски) получают `δ=0`.
4. **Сглаживаем** перенесённое поле `δ` (Laplacian δ-smoothing,
   `smooth_iters`/`smooth_alpha`) — UV-NN даёт ступеньки на швах зон.
5. **Сглаживаем перенесённые ГРУППЫ** (метки `gcid`/`zone`) —
   `smooth_transferred_groups=True`: majority-vote (мода в 1-кольце) за
   `group_smooth_iters` итераций (отдельно от δ-смуза!) убирает «крапинки» на
   швах. Дискретный аналог Laplacian-смуза; непокрытые вершины (`-1`) не
   меняются. Шум групп в 4-й UV-панели → крути `group_smooth_iters`; шум
   геометрии (3D deformed) → крути `smooth_iters`/`smooth_alpha`.
6. Открывается окно **FBX rest | deformed | deformed-smoothed**.

Дампы → `run_v6_*/fbx/`: `delta_transferred.csv`,
`delta_transferred_smoothed.csv`, `deformed_transferred.csv`,
`deformed_transferred_smoothed.csv`,
`clusters_transferred.csv` (`vertex_idx,zone,global_cluster_id,covered`).

> δ переносится напрямую: обе головы нормализованы `normalize_bbox` в один
> фрейм, поэтому масштаб деформаций сопоставим.

## Вспомогательные инструменты

### Expression Browser (`expression_browser.py`)
Интерактивный просмотр выражений FLAME: 3D-голова + слайдеры betas
(expr 300..399, shape 0..299). Крутишь → видишь мимику. Кнопка экспорта в OBJ
печатает активные betas (копируй в `Custom betas`).
```bash
python python/scripts/motion_groups_v6/expression_browser.py [--n-expr 30 --range 8]
```

### Spectral Descriptors (`spectral_descriptors.py`)
Две головы рядом (FLAME + FBX), справа выбор дескриптора **HKS / WKS** и его
настройки (k собственных функций, число каналов t/energy, WKS sigma). Кнопка
«Считать сигнатуры» красит обе головы по выбранному каналу (CMAP_HEAT). Shift+
click выбирает точку на каждой голове; маленькое окошко выводит значения
сигнатур в этих точках + ‖Δ‖ и cos-схожесть полных сигнатур. Дескрипторы
строятся из `compute_spectrum` (L·v=λ·M·v):
  • `HKS(x,t) = Σ exp(-t·λ_k)·φ_k(x)²`
  • `WKS(x,e) ∝ Σ φ_k(x)²·exp(-(e-log λ_k)²/2σ²)`
**Functional map** (кнопка «Построить карту FLAME→FBX»): по парам дескрипторов
строит матрицу C (k×k) в спектральном базисе (least squares + Тихонов), затем
точечное соответствие FBX→FLAME через NN в спектре (`Φ_dst` vs `Φ_src·Cᵀ`).
Цвет XYZ→RGB с FLAME переносится на FBX по карте: **одинаковый цвет = найденное
соответствие** (визуальная проверка качества карты). «k базиса для карты» —
размер C (меньше = глаже/устойчивее, больше = точнее).

**ZoomOut уточнение** (Melzi 2019, чекбокс, вкл. по умолчанию): сырой LS-map
«слипается» — много вершин FBX мапятся в горстку вершин FLAME (FBX выглядит
раскрашенным жёсткими зонами, а не гладким градиентом). ZoomOut итеративно
наращивает базис k→k_max, чередуя «карта → точечное соответствие → карта», и
резко повышает биективность. Статус показывает **покрытие** — % уникальных
вершин FLAME в соответствии (чем выше, тем плотнее/честнее карта). «Сброс карты»
возвращает обычную раскраску по каналу.

Открывается кнопкой **«🔬 Spectral descriptors»** в основном GUI (подхватывает
текущий FBX) или напрямую:
```bash
python python/scripts/motion_groups_v6/spectral_descriptors.py [--fbx <mesh> --n-eigs 120]
```

### Pipeline Viewer (`pipeline_viewer.py`)
До **4 окон в 3D**: 1 голова-референс FLAME + до **3 источников FBX**
(`--fbx` можно указать до 3 раз; в основном GUI — поля «FBX path», «FBX 2», «FBX
3»). Перенос выражения идёт на **все** источники одновременно; нижняя UV-полоса
показывает развёртки FLAME + всех источников (зоны и перенесённые группы).
Справа
конфигурация и кнопка **«Вперёд ▶»**, проводящая обе головы по шагам с
перекраской:
  0. Shift+click — точки тепла (сначала FLAME, потом FBX, одинаковое число);
  1. **single-t диффузия** (статично, без анимации) — суммарное тепло;
  2. **зоны** — multi-t enrichment + argmax-partition + маска по single-t reach;
  3. **кластеры** — motion на FLAME / heat+позиция на FBX;
  4. **перенос выражения** FLAME→FBX (UV-NN + барицентрика) — FBX деформируется,
     FLAME показывается со своей родной деформацией для сравнения.
Expr betas задаются в поле справа или флагом `--expr`. После переноса —
слайдеры **усиление деформации** и **Laplacian smooth** (реалтайм). Нижняя
полоса — UV-развёртка зон (FLAME / FBX / перенесённые группы); **W** включает
каркас в активном вьювере.

**Relax UV-островов** (после шага «зоны»): выбор метода + слайдер итераций +
кнопка **Relax** — релаксирует UV-острова обеих голов на месте:
  • **ARAP** — минимум искажений (Liu 2008), лучший по качеству;
  • **Laplacian** — сглаживание интерьера, граница фикс. (быстро убирает
    дёрганость/перехлёсты сетки);
  • **Spring** — mass-spring к 3D-длинам рёбер (изометрия рёбер).
После релаксации остров автоматически доворачивается по мировым осям (мировой
Y → +V, 2D Procrustes к world-flat эталону) — чтобы не «съезжал» с ориентации.

**⚡ Авто-перенос (весь пайплайн)** (кнопка): один клик прогоняет всё —
сброс → авто-анкоры (MediaPipe) → применить эмоцию → диффузия (+кроп если
включён) → зоны (+авто-relax) → кластеры → перенос. Использует текущие
настройки (betas, лендмарки, чекбоксы). Для ручного контроля — кнопка
«Вперёд ▶» по шагам.

**💾 Сохранить деформацию** (кнопка, после переноса): для каждого источника
пишет ФИНАЛЬНУЮ деформацию (с текущим усилением+smooth) в
`python/scripts/debug_output/deform_<ts>/<имя>`:
  • `.fbx` — деформированный меш (verts_rest + δ, через assimp; фолбэк `.obj`);
  • `.h5` — HDF5 per-vertex таблица: `vertex_index`, `direction` (единич.
    вектор сдвига), `magnitude` (|δ|, сила сдвига), + raw `delta`, `rest`,
    `deformed`, `faces`. Фолбэк CSV, если нет h5py.

**Нормализация по активной зоне** (всегда, после single-t диффузии): меш
выравнивается так, чтобы **активная зона** — вершины, где макс. нормированное
влияние любого anchor'а > `heat threshold` (объединение полигонов влияния тепла
всех анкоров) — вписывалась в bbox (центр + деление на диагональ). δ FLAME
масштабируется тем же фактором. Это происходит **независимо от чекбокса
обрезки** — bbox всегда считается по полигонам влияния.

**Обрезка по активной зоне** (чекбокс «Обрезать меш по активной зоне single-t
→ bbox»): дополнительно к нормализации **удаляет** неактивную геометрию
(оставляет только активные вершины + их грани, переиндексирует). Убирает лишнюю
геометрию (затылок/шея) до зон/переноса. anchors/heat переносятся. «Сброс»
восстанавливает исходный меш.

**Авто-анкоры (MediaPipe)** (кнопка на шаге 0): для каждой головы рендерим её
ортографически спереди (растеризация рейкастом, shade по нормали), прогоняем
**MediaPipe Face Mesh** (468/478 лендмарок, фикс. семантика), для выбранных
индексов (`auto_anchors.DEFAULT_LANDMARKS` — углы рта, веки, крылья носа, брови,
щёки, подбородок) пускаем тот же ортолуч обратно в меш → ближайшая вершина =
anchor. Рендер и рейкаст в одном ортокадре → пиксель и луч согласованы без
калибровки камеры. Перебирает виды (±Z, ±X), берёт где лицо задетектилось.
Требует `pip install mediapipe==0.10.14` (модель внутри пакета).

Открывается кнопкой **«▶ Pipeline viewer»** в основном GUI (подхватывает FBX и
Custom betas) или:
```bash
python python/scripts/motion_groups_v6/pipeline_viewer.py [--fbx <mesh> --expr "300:8.0"]
```

### Batch Transfer (`batch_transfer.py`)
Батч-перенос одной эмоции на **папку FBX-голов** → один HDF5-датасет (headless).
Для каждого FBX: авто-анкоры (MediaPipe) на FLAME и FBX, single-t диффузия →
multi-t зоны → кластеры, UV-перенос δ FLAME→FBX. Пишет в `data/dataset.h5`
(корень репо): на каждую голову своя группа `heads/<имя>` с `verts` (нейтраль),
`faces`, `delta` (перенесённая деформация). Пока одна эмоция на прогон.
В основном GUI: поле **«Папка FBX (батч)»** + кнопка **«📦 Батч-перенос…»**
(эмоция берётся из Custom betas, форма — из Shape preset). Или напрямую:
```bash
python python/scripts/motion_groups_v6/batch_transfer.py \
    --fbx-dir <папка> --expr "300:8.0" [--out data/dataset.h5 --shape "0:2.5"]
```
HDF5-схема: `heads/<имя>/{verts(N,3) нейтраль, faces(K,3), delta(N,3)}`,
атрибуты `expr`, `shape`, `n_heads`.

### Head Generator (`head_generator.py`)
GUI-генератор N случайных FLAME-голов для датасета. Сэмплит случайные **shape-
беты** (индексы 0..299, `N(0, std)`) и сохраняет каждую голову в `.fbx` (через
assimp; фолбэк `.obj`). Настройки:
  • **число голов** — сколько сгенерировать;
  • **сила различий — std (min)** — амплитуда случайных shape-бет (больше =
    головы сильнее отличаются по форме);
  • **диапазон std** (чекбокс + std max) — если включён, каждая голова получает
    СВОЙ разброс `s ~ uniform(std, std_max)` → непрерывный спектр «силы
    различий» (и почти-нейтральные, и сильно отличающиеся головы). Рекомендуется
    реалистичный диапазон ~0.5..1.5 (FLAME betas ≈ N(0,1) по реальным людям);
  • **число shape-компонент** (1..300) — сколько первых компонент варьируем
    (первые дают крупные изменения формы черепа/лица);
  • **seed** — воспроизводимость; **нормализовать в bbox**.
Пишет `manifest.csv` с betas каждой головы. Кнопка **«👁 Просмотр головы»**
открывает выбранный меш (FBX/OBJ/PLY) в окне Open3D (`mesh_viewer.py`,
подпроцесс). Открывается кнопкой **«🧬 Генератор голов»** в основном GUI или:
```bash
python python/scripts/motion_groups_v6/head_generator.py
# просмотр одного меша напрямую:
python python/scripts/motion_groups_v6/mesh_viewer.py <путь.fbx>
```
